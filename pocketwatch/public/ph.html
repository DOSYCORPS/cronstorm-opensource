What is Pocketwatch?

Recurring job scheduling as a service

The system is robust, fault-tolerant and scalable. Normally you can purchase either a subscription, or a one-off timer. But this is a free demo API key for Product Hunt readers to try it out without paying anything.

Pocketwatch provides Pocketwatch Timers which are your schedulers for recurring tasks that can issue HTTP requests at intervals as fine as 1 second, or longer.

You can use them to:

  • schedule recurring jobs
    e.g. POST /purge-cache every 15 minutes
  • schedule background tasks
    e.g. POST /check-status every 10 seconds
  • run jobs
    e.g. GET /tweet-web-crawl every 1 minute
  • trigger recurrent workloads
    e.g. PATCH /run-chromeless-tests every 5 minutes

all the things you might use cron/crontab for (or systemd timers for), but when you want it more scalable, more fault-tolerant than one machine running cron — you use Pocketwatch Timers.

Just fire and forget.

Just pick an interval (say, 3 seconds), and a duration (say 5 weeks), set your URL and HTTP method, and you're good to go.

Limitations of this demo

This demo uses an API key I set up specifically for Product Hunt readers. The key is i_am_ph_and_proud. To save resources, any timers created with this free demo key will only last roughly 2 weeks regardless of the maximum duration you put, and the key itself also has a maximum number of timers and hits. You can also try other HTTP methods, and include a body and Content-Type for the requests by using this key with the API directly.
